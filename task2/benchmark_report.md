## Выводы

### 1. Пропускная способность

На малых сообщениях (128B-1KB) Redis показал преимущество (1050 msg/s против 1000 msg/s у RabbitMQ). 
На больших сообщениях (10KB+) RabbitMQ значительно опережает Redis (636-1288 msg/s против 45-130 msg/s).
В среднем RabbitMQ показал на 72.7% выше пропускную способность.

### 2. Влияние размера сообщения

RabbitMQ лучше переносит увеличение размера сообщения:
- Redis при переходе с 1KB на 10KB падает в 23 раза (1050 → 45 msg/s)
- RabbitMQ сохраняет производительность на 10KB (1000 → 1265 msg/s)
- На 100KB RabbitMQ выдаёт 636-700 msg/s, Redis — только 125-130 msg/s

### 3. Точка деградации

| Брокер | Размер | Скорость | Признаки |
|--------|--------|----------|----------|
| Redis | 10 KB | любая | Скорость падает до 45 msg/s, не масштабируется |
| RabbitMQ | 100 KB | 1000+ msg/s | Очередь >4000, потери 1.3-2.4% |

### 4. Рекомендация по выбору инструмента

- **Мелкие сообщения, максимальная скорость** → Redis
- **Крупные сообщения, надёжность** → RabbitMQ
- **Смешанный сценарий (как в тесте)** → RabbitMQ (предсказуемое поведение)

В рамках данного эксперимента **RabbitMQ** показал лучшие результаты благодаря:
- Более высокой средней пропускной способности
- Устойчивости к росту размера сообщений
- Плавной деградации под нагрузкой
